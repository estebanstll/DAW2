# Security Policy

## Purpose

This document explains how to report security vulnerabilities related to this repository.

If you discover a security issue, please do not open a public issue.

---

## Supported Versions

Support is provided for the current version available in the `main` branch.  
Security fixes will be applied to the active version of the project.

---

---

## Process

1. The vulnerability will be evaluated.
2. Receipt of the report will be confirmed.
3. A fix will be developed and tested.
4. The correction will be published.

---

## Good Practices

- Do not commit credentials or private keys to the repository.
- Use `.env` files for sensitive variables.
- Keep dependencies up to date.
